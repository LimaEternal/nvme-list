# FIO-Test — Автоматический бенчмаркинг накопителей

Автономный скрипт для автоматического тестирования производительности несистемных дисков на сервере с помощью утилиты `fio`. Скрипт сам определяет типы подключённых дисков (NVMe/SAS/SATA), подбирает оптимальные параметры тестирования и выводит результаты в реальном времени.

## Структура проекта

```
fio-test.py              # Главный скрипт (точка входа)
configs/
  __init__.py
  nvme.py                # FIO-конфиги для NVMe (bs=128k, 8 потоков)
  sas.py                 # FIO-конфиги для SAS (bs=64k, 2 потока)
  sata.py                # FIO-конфиги для SATA (bs=64k, 1 поток)
utils/
  __init__.py
  scanner.py             # Определение дисков через lsblk
  reporter.py            # Генерация MD-отчёта
nvk/                     # Рабочие файлы наставника (архив)
tools/                   # Утилиты для обработки чатов
```

## Зависимости

- Python 3.10+
- `rich` — красивый вывод в консоль (таблицы, прогресс-бары)
- `fio` — утилита тестирования дисковой производительности (должна быть установлена в системе)

Установка Python-зависимостей:

```bash
pip install rich
```

## Запуск

### Базовое тестирование

```bash
python fio-test.py
```

Скрипт сканирует систему, находит все несистемные диски, запускает 4 теста на каждом (seq read/write, random read/write 4k) и выводит результаты в консоль + MD-отчёт.

### С прекондишнингом

```bash
python fio-test.py --precond
```

Перед тестами выполнится запись 100% объёма каждого диска (bs=1M) для стабилизации производительности SSD. **Все данные на дисках будут уничтожены.**

### Свой путь для отчёта

```bash
python fio-test.py --output /path/to/report.md
```

По умолчанию отчёт сохраняется как `fio_report_<дата_время>.md` в текущей директории.

### Изменение длительности теста

```bash
python fio-test.py --runtime 60
```

Каждый тест будет длиться 60 секунд вместо 30.

### Без запроса подтверждения (для автоматизации)

```bash
python fio-test.py --yes
```

Пропускает интерактивный запрос подтверждения. Удобно для запуска в скриптах или cron.

### Все флаги вместе

```bash
python fio-test.py --precond --runtime 60 --output result.md --yes
```

## Безопасность

**Критически важно:** скрипт работает напрямую с блочными устройствами (`/dev/*`). Тесты записи (`seq_write`, `rand_write`) перезаписывают данные на дисках. Восстановление невозможно.

Перед запуском скрипт:
1. Показывает список обнаруженных дисков
2. Выводит красное предупреждение об уничтожении данных
3. Запрашивает подтверждение: введите `y` для запуска или любую другую клавишу для отмены

Системные диски (с точкой монтирования `/`) автоматически исключаются из тестирования.

## Какие тесты выполняются

| Тест | Режим | Блок | Потоки | Для чего |
|------|-------|------|--------|----------|
| seq_read | Последовательное чтение | 128k (NVMe) / 64k (SAS/SATA) | 1 | Максимальная пропускная способность |
| seq_write | Последовательная запись | 128k / 64k | 1 | Максимальная пропускная способность на запись |
| rand_read | Случайное чтение 4k | 4k | 8 (NVMe) / 2 (SAS) / 1 (SATA) | Максимальные IOPS |
| rand_write | Случайная запись 4k | 4k | 8 / 2 / 1 | Максимальные IOPS на запись |

Параметры подбираются автоматически по типу интерфейса:
- **NVMe** — агрессивный профиль, 8 параллельных потоков (QD256)
- **SAS** — серверный профиль, 2 потока (QD64)
- **SATA** — строго 1 поток (QD32),AHCI не переваривает многопоточность

Для дисков с физическим сектором 4096 (4Kn) размер блока автоматически корректируется.

## Результаты

### Консоль

Во время тестирования в консоли отображается интерактивная таблица (библиотека `rich`):
- Текущий тест помечен значком ⏳
- Завершённые тесты — ✅ с реальными цифрами
- Прогресс-бар показывает общий ход выполнения

### MD-отчёт

После завершения создаётся Markdown-файл с таблицами:
- Список обнаруженных дисков (модель, серийник, интерфейс, объём)
- Результаты тестов для каждого диска (IOPS, МБ/с, Latency avg, Latency p99)
